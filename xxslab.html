The Project Story

The XSS Live Lab was engineered to provide a visual bridge between offensive payloads and defensive coding. It focuses on DOM-based XSS, the most common vector in modern JavaScript-heavy applications. By providing a real-time playground, I bridge the conceptual understanding of vulnerabilities with their actual execution on the Document Object Model.

Architecture

The lab uses a vanilla JavaScript engine that toggles between innerHTML and textContent nodes. This side-by-side simulation proves that security is often just a matter of choosing the correct API for data rendering. The analysis sidebar is a custom-built event listener that interprets the loaded payload and serves a curated security report.

The Trenches

The biggest technical hurdle was creating a "controlled explosion." We had to ensure that when a user executes a "UI Defacement" script, it only affects the simulated Victim Window and doesn't crash the entire portfolio's runtime. It required strict scoping and precise event handling to keep the lab dangerous yet perfectly contained.
